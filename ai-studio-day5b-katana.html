The Swordsmith's Rule

Study the blade before you strike. A swordsmith doesn't guess how a katana fits together — they learn the anatomy first. The tang, the guard slot, the handle socket. Every dimension serves a purpose. When you understand how the real thing works, the code writes itself. When you guess, you get pieces that don't fit.

Before building anything, you need to understand how a katana actually fits together. Not a guess — the real anatomy. Tell Claude Code to build you a study model.

💬 Say this to Claude Code
I want to build a 3D-printable katana in three pieces: blade, guard, and handle. Before we build, research how a real katana's parts connect. Specifically: what is the tang (nakago) and how does it pass through the guard (tsuba) into the handle (tsuka)? What shape is the tang cross-section? What is the sori (blade curve) and which direction does it curve? What does the blade look like in cross-section (the shinogi ridge)?

Then build me a study model in OpenSCAD with these requirements:

1. The blade has a flat rectangular tang extending down from its base. The tang passes THROUGH a matching slot in the guard, and INTO a blind socket in the top of the handle. One continuous connection — no separate tabs.
2. The guard is an elliptical disc with a rectangular through-slot.
3. The handle has a rectangular socket (blind hole) in its top to receive the tang.
4. Use 0.3mm clearance per side on all joints for 3D printing.
5. The blade should have proper sori (curve) — the spine curves away from the cutting edge. The curve should be visible from the flat face of the blade.
6. The blade cross-section should be a flattened hexagon (shinogi-zukuri), not a rectangle — sharp edge on one side, flat spine on the other.
7. Add an "explode" parameter (0 = assembled, 1 = fully separated) so I can see the pieces apart.

Export each piece as a separate STL, then create an HTML viewer I can open in my browser with a slider to control the explode amount. Color the blade silver, guard bronze, handle brown.

Open the viewer. Use the explode slider to pull the pieces apart. This is where you learn the anatomy:

BLADE The curved steel. Has a sharp edge, a flat spine, and a ridge line (shinogi) running down the middle. TANG The rectangular bar extending down from the blade base. This is the key — it passes through the guard and locks into the handle. GUARD The flat disc (tsuba). Has a rectangular slot that the tang passes through. Sits between blade and handle. HANDLE The grip (tsuka). Has a rectangular socket in the top that receives the tang. ASSEMBLY: tang slides through guard slot, into handle socket. Guard is sandwiched between blade base and handle top.
  • 🔌 Assembled (slider at 0%): The three pieces should sit together cleanly. Tang hidden inside. No gaps.
  • 🔄 Exploded (slide right): You should see the tang hanging from the blade, the guard's slot, and the handle's socket. All three shapes match.
  • ⚔️ The curve: Looking at the blade from the flat side, the spine should arc gently. That's the sori — it's what makes a katana a katana.
🔨
Why OpenSCAD? This session uses a code-based modeling tool. Every shape, every joint, every dimension is defined in code — Claude Code writes it, you direct it. The advantage: joints are precise, dimensions are exact, and every piece is guaranteed solid and watertight. You'll see the result in an interactive 3D viewer you can rotate and zoom.
💡
Why research first? Without real katana anatomy front-loaded, the AI will guess — and it'll get the joints wrong. It might put two tabs colliding inside the guard, or curve the blade in the wrong direction. The research step is what makes the difference between a model that looks right and one that actually works.
  1. Give Claude Code the full prompt above. It will research, build the model, and create the viewer.
  2. Open the viewer in your browser. Drag to rotate, scroll to zoom.
  3. Use the explode slider. Study how the tang connects through the guard into the handle.
  4. Look at the blade curve from the flat side. Does it look like a katana? If not, tell Claude Code what's wrong.

You have a working katana. Now change it. Every part of the design is a parameter you can adjust. The rule: change one thing at a time, check the viewer, then change the next thing.

Pick any of these to customize:

  • ⚔️ Guard shape: It's elliptical right now. Make it circular, square, hexagonal, or star-shaped. Change the size. This is the most visible customization.
  • 🌌 Blade curve: More sori (dramatic curve) or less (straighter, more like a European sword). Your call.
  • 📏 Blade length: Longer for a full katana, shorter for a tanto (dagger).
  • 🔨 Handle shape: It's octagonal right now. Make it round, hexagonal, or squared. Change the length. Add a bigger pommel.
💬 Example: change the guard
Change the guard from an ellipse to a [SHAPE YOU WANT] shape, about [SIZE]mm across. Keep the tang slot exactly the same — same dimensions, same clearance. Rebuild the model and update the viewer.
Replace [SHAPE] and [SIZE] with your choice. The key phrase is "keep the tang slot exactly the same" — that protects the joint.
💬 Example: change the blade curve
Make the blade curve more dramatic — increase the sori so it's really visible. Keep the same cross-section shape and the same tang. Update the viewer.
💬 Example: change the handle
Make the handle [round / hexagonal / square] instead of octagonal. Make it [longer / shorter]. Keep the socket dimensions exactly the same. Update the viewer.
⚠️
The golden rule of customization: change the look, but never change the joint dimensions. The tang, the guard slot, and the handle socket all need to match. If you change one, you break the connection. Always say "keep the tang slot exactly the same" or "keep the socket dimensions the same."

Your katana design is done. Before exporting the full pieces, you validate the trickiest part: the joint.

Step 1: Test the joint alone.

💬 Say this to Claude Code
Build me a joint test piece — just the last 20mm of the blade (with the tang), the guard, and the top 20mm of the handle (with the socket). No need for the full blade or full handle — just the area where they connect. Export these test pieces as STLs and create a viewer with the explode slider. I want to inspect the joint up close.

Open the test viewer. This is a zoomed-in look at just the joint area. Check everything:

  • 🔌 Tang shape: Does the rectangular tang match the guard slot and handle socket?
  • 📏 Clearance: Is the tang slightly smaller than the slot? (It should be — that's the 0.3mm gap for printing.)
  • 🔄 Assembly direction: Could you slide these together with your hands? Tang goes down through guard, into handle.
  • ⚔️ No overlap: When assembled, does the guard sit flat between blade base and handle top? No pieces going through each other?

If anything looks wrong, tell Claude Code exactly what's off. Fix it before moving on.

💡
This is how real engineers work. In manufacturing, you always test the joint before building the full product. A 2-minute check on a small test piece saves hours of reprinting. Changing code is free. Reprinting costs time and filament.

Step 2: Export the full pieces.

Joint looks good? Now export the complete katana.

💬 Say this to Claude Code
The joint looks good. Now export the full katana — each piece as its own STL file: katana-blade.stl, katana-guard.stl, katana-handle.stl. Run analyze_model on each piece to check for geometry problems. Then create a final HTML viewer with all three pieces and the explode slider — this is my final assembly view.

Open the final viewer. This is your last check before printing. Toggle the explode slider one more time — every joint should show matching shapes. In assembled view, the katana should look complete.

// troubleshooting
Problem: tang doesn't match the slot

Tell Claude Code: "The tang and slot don't match — show me the exact dimensions of the tang, the guard slot, and the handle socket. They should all be the same rectangle with 0.3mm clearance."

Problem: pieces overlap or go through each other

Tell Claude Code: "The pieces are overlapping in the assembled view. Make sure the guard sits between the blade base and the handle top with no intersection. Show me a cross-section view."

Problem: blade doesn't look like a katana

Tell Claude Code: "The blade doesn't look right. Research what a real katana blade looks like — the sori curve direction, the shinogi cross-section, the kissaki tip shape — and rebuild the blade based on the real geometry."

The key move

When something looks wrong, describe what you see ("the tab is sticking out above the guard") not what you think the fix is. Claude Code will figure out the fix — your job is the eyes.
