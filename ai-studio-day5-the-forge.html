The One Rule

A 3D printer builds from the bottom up, layer by layer. It can't print in mid-air. So there's one rule: nothing overhangs more than 45°. Think of stacking wet sand — you can slope the sides, but you can't build a shelf sticking straight out. That's the design challenge today: make something that looks great and respects gravity.

Research
Teach the Machine
10 min
›

Before you build anything, do something most people don't think to do: tell Claude Code to learn the domain first. It'll research how to make Three.js models that actually work for 3D printing — then it builds smarter.

💬 Say this to Claude Code
I want to make a 3D model using Three.js, export it as an STL file, and 3D print it — without needing any supports. Before we start, research what you need to know about making this work: Three.js to STL pipeline, printable geometry, and designing for no supports. Tell me the key things I should understand.

Claude Code will come back with a summary. The most important things it'll tell you:

  • ▲ The 45° rule: No surface can overhang more than 45° or the printer has nothing to build on.
  • 💧 Watertight mesh: The model has to be a sealed shell with no holes. Think: if you filled it with water, none would leak out.
💡
The technique here: You just told AI to educate itself before starting work. That's a real skill — Claude Code builds better when it understands the problem first. You'll use this technique for the rest of your life.
  1. Open Claude Code and give it the research prompt.
  2. Read what it finds. The 45° rule is the big one.
  3. Ask any questions you have — better to understand now than during the build.

Write Your Design Spec
Answer these three questions:

→ What is it? (name and description)
→ What makes it interesting? (the detail, the shape, the function)
→ Where might overhangs be? (and how will you handle them)
✅
Validate your idea. Once you've written your spec, run it past Claude Code: "Here's what I want to build. Given the 45-degree rule and support-free constraint, will this design work? What parts might be problematic, and how would you modify them?" Let it sanity-check your design before you commit.
// example — design specs
War Helmet

What: A Spartan helmet on a small display base, roughly 60mm tall. The crest goes straight back, T-shaped face opening, cheek guards flare out.

Interesting: The crest is the showpiece — thick, horsehair-style, running front to back. The face opening gives it that iconic menacing look. I want it to feel like a museum artifact.

Overhangs: The cheek guards extend down and slightly outward — but they angle less than 45° since they follow the helmet curve. The crest sits on top. The base supports everything. Should work.

Crystal Formation

What: A cluster of 5-7 hexagonal crystal shafts growing from a rough base, tallest one ~80mm.

Interesting: Each crystal is a slightly different height and angle, like a real mineral specimen. Some are thick, some are thin. The base looks like raw rock.

Overhangs: The tilted crystals could be a problem — I'll keep the tilt angles modest. The pointed tips taper naturally. The rock base builds upward. Should be fine.

  1. Look at the model grid. What would be satisfying to hold in your hand?
  2. Write your design spec: what it is, what makes it interesting, where overhangs might be.
  3. Run it past Claude Code for a feasibility check before building.

Phase 03 — Build
03
Build
Model It
25 min
›

Same loop you used for game building — build, look at it, fix one thing, repeat — but now you're sculpting a 3D object.

💬 Say this to Claude Code
Build me a 3D model of [YOUR DESIGN] using Three.js. Show it to me in the 3D viewer so I can rotate it. It needs to be 3D-printable without supports — keep everything within the 45° overhang rule, make the mesh watertight, use at least 2mm wall thickness, and give it a flat base for the print bed. Make it look good — this is going on a shelf.
Replace [YOUR DESIGN] with your design spec from Mission 2.

Claude Code will show you the model in a 3D viewer you can rotate with your mouse. Spin it around. Look at it from below — that's where overhang problems hide.

🔁
One change at a time. Same rule as game building. Fix one thing, rotate, inspect. Then fix the next thing. Don't stack five requests at once.
// example — building a spartan helmet
Round 1 — First Model

You: Build me a 3D model of a Spartan helmet on a small display base. Crest goes straight back, T-shaped face opening, cheek guards flare out. Keep it within 45° overhangs, watertight mesh, 2mm walls, flat base. Show it in the 3D viewer.

Claude Code: [writes Three.js code, shows interactive 3D model]

Round 2 — First Fix

You (after rotating): The crest looks too thin and flat. I want it thicker and taller — like a mohawk of horsehair, not a fin. Also the cheek guards are too short, they should come down to the chin.

Claude Code: [adjusts crest and cheek guards, shows updated model]

Round 3 — Overhang Check

You: I looked at it from below. The cheek guards — their undersides look like they might be past 45 degrees. Can you check the actual angles and adjust if needed?

Claude Code: [calculates angles, adjusts the cheek guards to 40°, shows updated model]

The loop: view → rotate → identify the problem → describe the fix → view again. 3-5 rounds is usually enough to get something you're proud of.

When you're happy with how it looks, ask Claude Code: "Will this print cleanly without supports? Check the overhangs, wall thickness, and mesh. Anything I should fix?" Let it do the engineering audit for you.

  1. Give Claude Code your design spec with the build prompt.
  2. When the 3D model appears, rotate it. Look at it from every angle, especially from below.
  3. Tell Claude Code one thing to change. View the updated model.
  4. Repeat until you're happy. 3-4 rounds is usually enough.
  5. Ask Claude Code to do a final printability check.

Phase 04 — Print
04
Export
From Screen to Hand
15 min + print time
›

Your model is done on screen. Now get it out of the browser and into your printer.

💬 Say this to Claude Code
Create an HTML file I can open in my browser that shows my model (with rotation controls) and has an "Export STL" button that downloads it as a .stl file. Use the Three.js STLExporter. Make sure 1 unit = 1 millimeter.

Open the HTML file in your browser, give it one last spin, and hit the export button. Then open the .stl in your slicer.

  1. Open the STL in your slicer (Cura, PrusaSlicer, BambuStudio — whatever you use).
  2. Preview it with supports turned off. If the slicer shows problem areas, go back to Claude Code and tell it what to fix.
  3. When it looks clean — print it. Watch the first few layers, then let it run.
📐
If the model looks tiny or huge in the slicer: STL files don't store units. Most slicers default to millimeters. If the scale is off, tell Claude Code to adjust and re-export.

Once you've printed it...
Level up ideas: Print multiple pieces that snap together. Design interlocking parts. Create a chess set where every piece is your own design. Build a model with moving parts — gears that actually mesh, hinges that actually rotate. Or go parametric: change one number in the code and the entire shape transforms. The pipeline you just learned — code to screen to slicer to printer — is the same one used in actual product design and manufacturing. You just did it.
