Mission 01 — Pick Your Object
01
Design
What Will You Hold?
5 min
›

The question is simple: what would be cool to hold in your hand? Something you designed, that didn't exist before today. Pick from the grid or invent your own.

⚔️
War Helmet
A Spartan, Viking, or samurai helmet — miniature scale, sitting on a small display base. Crest, face guard, the works.
🏰
Fortress Tower
A medieval tower with battlements, arrow slits, and a spiral staircase inside. Could be the start of a whole castle.
♟
Custom Chess Piece
Design your own king, queen, or knight. Not the standard shape — YOUR version. A dragon for the knight? A throne for the king? You decide.
👹
Dragon Skull
A low-poly dragon skull on a mounting base. Horns swept back, jaw angled up. The kind of thing you'd see in a fantasy museum.
🛡️
Shield
A Greek aspis, a Roman scutum, or a Viking round shield. Add an emblem on the front — your own design or a historical one.
💎
Crystal Formation
Hexagonal crystals erupting from a rocky base at different heights and angles — like a real mineral specimen for your desk.
🌇
Miniature Cityscape
A tiny skyline rising from a base — towers, rooftops, antenna spires. Your own city that fits in your palm.
💡
Your Own Idea
A trophy, a game token, a rocket, a mountain terrain, a dice tower, a character bust — if you can describe it, Claude Code can build it.
Write This Down
In one or two sentences: what is it, and what makes it interesting?

Example: "A Spartan helmet on a small base. The crest goes straight back, there's a T-shaped opening for the face, and the cheek guards flare out."
💡
Keep it solid. Objects that build upward from a base print the cleanest. A helmet on a stand, a tower rising from the ground, a chess piece on its base — all great. A flying dragon with outstretched wings? That's a fight with gravity you don't need today.
  1. Look at the grid. Which one would you actually want on your desk?
  2. Write your idea in one or two sentences. Be specific about what makes it yours.

Mission 02 — Build It
02
Build
From Idea to 3D Model
30 min
›

This mission has two parts. First, you'll tell Claude Code to learn how to make 3D-printable models. Then you'll build yours. This order matters — Claude Code builds much better geometry when it understands the constraints first.

💬 Step 1 — Say this first
I want to make a 3D model using Three.js that I can export as an STL file and 3D print without any supports. Before we start building, research what you need to know to make this work: the Three.js to STL pipeline, what makes geometry 3D-printable, and how to design for printing without supports. Tell me the key rules.

Claude Code will come back with what it learned. The most important things:

▲
The 45° rule. A 3D printer builds from the bottom up, layer by layer. It can't print in mid-air. So nothing should overhang more than 45° — think of stacking wet sand. You can slope the sides, but you can't build a shelf sticking straight out.
💧
Watertight mesh. The model has to be a sealed solid shell — no holes, no gaps. If you could fill it with water, none should leak out. Claude Code knows how to do this, but only if you tell it to.

Once Claude Code has done its research, give it your build request:

💬 Step 2 — Then say this
Now build me a 3D model of [YOUR IDEA]. Show it to me in the 3D viewer so I can spin it around. Apply everything you just learned — watertight mesh, no overhangs past 45°, minimum 2mm wall thickness, flat base for the print bed. Make it look good — this is going on my shelf.
Replace [YOUR IDEA] with your description from Mission 1. If Claude Code asks clarifying questions, answer them — that means it's trying to get the details right.

Claude Code will show you the model in a 3D viewer you can rotate with your mouse. Spin it around. Look at it from every angle. Look at it from below — that's where overhang problems hide.

🔁
One change at a time. Same rule as game building. "Make the horns longer" — rotate, check. "The base is too thin" — rotate, check. Don't stack five requests. Each round the model gets closer to what you imagined.
// example — building a spartan helmet
Round 1 — Research

You: I want to make a 3D model using Three.js that I can export as an STL and 3D print without supports. Research what you need to know to make this work. Tell me the key rules.

Claude Code: [researches, comes back with: 45° overhang rule, watertight mesh, minimum wall thickness, flat base, manifold geometry, etc.]

Round 2 — First Model

You: Now build a Spartan helmet on a small display base. Crest goes straight back, T-shaped face opening, cheek guards flare out. Apply everything you just learned. Show it in the 3D viewer.

Claude Code: [builds with proper wall thickness, sealed mesh, no overhangs — shows interactive 3D model]

Round 3 — First Fix

You (after rotating): The crest looks too thin and flat. I want it thicker and taller — like a mohawk of horsehair, not a fin. Also the cheek guards are too short, they should come down to the chin.

Claude Code: [adjusts crest and cheek guards, shows updated model]

Round 4 — Getting It Right

You: Way better. But the base looks like a plain disc. Can you make it look like a rough stone pedestal — slightly irregular edges, like it belongs in a museum?

Claude Code: [adds irregular stone-like base geometry]

That's the flow. Research first, then build, then iterate. The research step is what makes the first model actually printable instead of just pretty on screen.

When you're happy with how it looks, ask Claude Code: "Will this print cleanly without supports? Check the overhangs, wall thickness, and mesh. Anything I should fix?" Let it do the engineering audit.

  1. Give Claude Code the research prompt. Read what it finds — the 45° rule is the big one.
  2. Give it your build request. When the model appears, rotate it from every angle.
  3. Tell Claude Code one thing to change. View the updated model.
  4. Repeat 3-4 rounds until you're happy with it.
  5. Ask Claude Code to confirm it'll print clean — overhangs, wall thickness, mesh integrity.

Mission 03 — Print It
03
Export
From Screen to Hand
15 min + print time
›

Your model looks good on screen. Now get it out of the browser and into your printer.

💬 Say this to Claude Code
Create an HTML file I can open in my browser that shows my model with rotation controls and has an "Export STL" button that downloads it as a .stl file. Make sure 1 unit = 1 millimeter.

Open the HTML file, give it one last spin, and hit the export button. Then open the .stl in your slicer.

  1. Open the STL in your slicer (Cura, PrusaSlicer, BambuStudio — whatever you use).
  2. Preview it with supports turned off. If the slicer shows problem areas, go back to Claude Code and tell it what to fix.
  3. If the model looks tiny or huge, tell Claude Code to adjust the scale and re-export. STL files don't store units, so this is normal.
  4. When it looks clean — print it. Watch the first few layers, then let it run.
🏆
